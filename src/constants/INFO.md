# Constants (Константи)

## Призначення папки

Папка містить константи та значення за замовчуванням, які використовуються по всьому додатку. Централізоване зберігання для легкого змінення налаштувань.

## Структура

```
src/constants/
├── defaults.ts           # Значення за замовчуванням (години, логотипи)
├── countries.ts          # Список країн для вибору
└── italianProvinces.ts  # Список італійських провінцій
```

## Файли

### defaults.ts

**Призначення:** Значення за замовчуванням для системи (години заїзду/виїзду, логотипи готелів).

**Що містить:**

1. **Логотипи готелів:**
   - `DEFAULT_HOTEL_LOGO_VARIANTS` - масив варіантів логотипів
   - `DEFAULT_HOTEL_LOGO_URL` - логотип за замовчуванням
   - `getRandomHotelLogo()` - функція для отримання випадкового логотипу

2. **Години заїзду/виїзду:**
   - `DEFAULT_CHECK_IN_HOUR` - година заїзду за замовчуванням (14:00)
   - `DEFAULT_CHECK_OUT_HOUR` - година виїзду за замовчуванням (10:00)

**Використання:**

- При створенні нового готелю (адміна)
- Якщо не вказано інші значення
- Для генерації випадкових логотипів

**Важливо:**

- Зміни в цьому файлі мають синхронізуватися з `admin-frontend/src/constants/defaults.ts`
- Значення використовуються в entity Admin як default

### countries.ts

**Призначення:** Список країн для вибору при створенні/редагуванні готелю.

**Що містить:**

- Масив об'єктів з кодом країни та назвою
- Використовується для випадаючого списку країн

**Використання:**

- В формах створення/редагування готелю
- Для валідації коду країни
- Синхронізовано з `admin-frontend/src/constants/countries.ts`

### italianProvinces.ts

**Призначення:** Список італійських провінцій для вибору при створенні/редагуванні готелю.

**Що містить:**

- Масив об'єктів з кодом провінції та назвою
- Використовується для випадаючого списку провінцій

**Використання:**

- В формах створення/редагування готелю
- Для валідації провінції
- Синхронізовано з `admin-frontend/src/constants/italianProvinces.ts`

## Важливі моменти

- **Синхронізація** - константи мають бути синхронізовані з фронтендом
- **Централізація** - всі значення за замовчуванням в одному місці
- **Легка зміна** - можна швидко змінити налаштування для всієї системи
- **Документація** - кожна константа має коментар з поясненням
