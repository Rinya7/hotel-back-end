# Seeds (Початкові дані)

## Призначення папки

Папка містить скрипти для створення початкових даних в базі даних (seed data). Використовується для ініціалізації системи з базовими даними.

## Структура

```
src/seeds/
└── seedSuperadmin.ts     # Створення супер-адміна
```

## Файли

### seedSuperadmin.ts

**Призначення:** Створення початкового супер-адміна для системи.

**Що робить:**

1. Ініціалізує підключення до БД
2. Застосовує всі міграції
3. Перевіряє чи існує superadmin
4. Якщо не існує - створює нового з даними з .env
5. Якщо існує - оновлює пароль (якщо змінився)

**Налаштування через ENV:**

- `SUPERADMIN_USERNAME` - ім'я користувача (за замовчуванням: "superadmin")
- `SUPERADMIN_PASSWORD` - пароль (обов'язково)

**Використання:**

```bash
npm run seed:superadmin
```

**Що створює:**

- Користувача з роллю `superadmin`
- Хешований пароль (bcrypt)
- Базові налаштування готелю (якщо потрібно)

**Безпека:**

- Пароль хешується через bcrypt
- Не зберігається в plain text
- Можна оновити пароль, змінивши в .env та перезапустивши seed

**Важливо:**

- Запускається тільки один раз при першому налаштуванні
- Можна запускати повторно для оновлення пароля
- Не видаляє існуючого superadmin, тільки оновлює

## Майбутні seeds

Можуть бути додані інші seed скрипти:
- `seedTestData.ts` - тестові дані для розробки
- `seedRooms.ts` - приклад кімнат
- `seedStays.ts` - приклад проживань
