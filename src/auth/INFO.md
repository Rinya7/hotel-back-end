# Auth (Аутентифікація та ролі)

## Призначення папки

Папка містить константи та типи для системи ролей користувачів. Централізоване визначення ролей для уникнення "магічних" рядків по всьому коду.

## Структура

```
src/auth/
└── roles.ts          # Константи ролей та типи
```

## Файли

### roles.ts

**Призначення:** Централізоване визначення ролей користувачів та допоміжні функції.

**Що містить:**

1. **Константи ролей:**
   ```typescript
   export const ROLES = {
     SUPER: "superadmin",
     ADMIN: "admin",
     EDITOR: "editor",
   } as const;
   ```

2. **Тип Role:**
   ```typescript
   export type Role = (typeof ROLES)[keyof typeof ROLES];
   // Результат: "superadmin" | "admin" | "editor"
   ```

3. **Допоміжні функції:**
   - `isSuperAdmin(role: Role): boolean` - перевірка чи є superadmin
   - `isAdmin(role: Role): boolean` - перевірка чи є admin

**Використання:**

- Імпортується в контролерах для перевірки ролей
- Використовується в middleware для авторизації
- Використовується в типах для типізації

**Приклад використання:**

```typescript
import { ROLES, Role } from "../auth/roles";

if (user.role === ROLES.SUPER) {
  // Логіка для superadmin
}
```

## Важливі моменти

- **Не використовувати "магічні" рядки** - завжди використовувати константи з ROLES
- **Типізація** - тип Role автоматично виводиться з констант
- **Централізація** - всі зміни ролей робляться в одному місці
